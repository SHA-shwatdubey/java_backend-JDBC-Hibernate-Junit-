QUICK REFERENCE GUIDE - Spring Hibernate Employee Management System
===================================================================

HOW TO RUN THE APPLICATION:
============================
1. Ensure PostgreSQL is running on localhost:5432
2. Create database: mydb
3. Build project: mvn clean install
4. Run Main.java as Java Application

EMPLOYEE ENTITY ATTRIBUTES:
============================
- id: Integer (Auto-generated Primary Key)
- name: String (Required, 100 chars max)
- email: String (Required, Unique, 100 chars max)
- salary: Double (Required)
- position: String (Optional, 100 chars max)
- department: String (Optional, 100 chars max)

EMPLOYEE CONTROLLER METHODS:
============================
1. addNewEmployee(Employee employee)
   - Adds a new employee to the database
   - Validates employee data

2. updateEmployeeDetails(Employee employee)
   - Updates existing employee information
   - Requires valid employee ID

3. deleteEmployee(Integer id)
   - Removes employee from database
   - Throws ResourceNotFoundException if ID not found

4. retrieveEmployee(Integer id)
   - Fetches single employee by ID
   - Returns Employee object

5. retrieveAllEmployees()
   - Fetches all employees from database
   - Returns List<Employee>

6. retrieveEmployeesByDepartment(String department)
   - Fetches employees by specific department
   - Returns List<Employee>

TRANSACTION MANAGEMENT:
=======================
- All database operations are wrapped with @Transactional
- Transaction Manager: JpaTransactionManager
- Transaction Isolation: Default
- Rollback on: RuntimeException

VALIDATION RULES:
==================
- Name: Required, minimum 2 characters
- Email: Required, valid email format (contains @)
- Salary: Required, must be non-negative
- Age (for User): Required, 0-150 range
- ID: Required for update/delete, must be positive

EXCEPTION HANDLING:
===================
- InvalidDataException: Thrown when data validation fails
- ResourceNotFoundException: Thrown when employee ID not found
- Both are RuntimeExceptions for Spring automatic rollback

DATABASE TABLES:
================
1. employees
   - id (INTEGER, PRIMARY KEY, AUTO_INCREMENT)
   - name (VARCHAR(100), NOT NULL)
   - email (VARCHAR(100), NOT NULL, UNIQUE)
   - salary (DOUBLE PRECISION, NOT NULL)
   - position (VARCHAR(100))
   - department (VARCHAR(100))

2. users
   - id (INTEGER, PRIMARY KEY, AUTO_INCREMENT)
   - name (VARCHAR(100), NOT NULL)
   - email (VARCHAR(100), NOT NULL, UNIQUE)
   - age (INTEGER)

LOGGING CONFIGURATION:
======================
- Logger: SLF4J with Logback
- SQL Logging: Enabled (hibernate.show_sql=true)
- SQL Formatting: Enabled (hibernate.format_sql=true)
- Statistics: Disabled

SPRING BEANS REGISTERED:
========================
1. dataSource - PostgreSQL DataSource
2. entityManagerFactory - JPA EntityManagerFactory
3. transactionManager - JPA Transaction Manager
4. employeeDAO - Employee Data Access Object
5. employeeService - Employee Business Logic Service
6. employeeController - Employee Request Handler
7. appService - Application Service
8. sessionFactory - Hibernate SessionFactory (legacy)

IMPORTANT NOTES:
================
✓ Use @Transactional for write operations (insert, update, delete)
✓ EntityManager is thread-safe when used within transactions
✓ Always validate data before persistence
✓ Use appropriate exception handling in service layer
✓ Component scanning is enabled for automatic bean detection
✓ XML configuration handles bean definitions for controllers and DAOs

EXAMPLE USAGE:
==============
// Create Employee
Employee emp = new Employee("John Doe", "john@example.com", 50000.0, "Developer", "IT");
controller.addNewEmployee(emp);

// Get All Employees
List<Employee> employees = controller.retrieveAllEmployees();

// Get by Department
List<Employee> itDept = controller.retrieveEmployeesByDepartment("IT");

// Update Employee
emp.setSalary(55000.0);
controller.updateEmployeeDetails(emp);

// Delete Employee
controller.deleteEmployee(emp.getId());
